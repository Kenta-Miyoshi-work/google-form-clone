# Google Form Clone Mock

Googleフォーム風のフォーム作成・回答管理モックです。バックエンドAPIには接続せず、画面仕様書や画面遷移図を作るために、フォーム作成から公開、回答、通知、監査ログ、エラー状態までをフロントだけで確認できるようにしています。

## 起動方法

```bash
npm install
npm run dev
```

Viteの表示するURLをブラウザで開きます。ログイン画面はデモ実装のため、任意のメールアドレスとパスワードで進めます。

## 主な画面

- ログイン画面: メールアドレス入力、パスワード入力の2段階ログインモック。
- 管理トップ: テンプレート一覧、ホバー時プレビュー、回答中/作成済みタブ。
- フォーム作成: 質問追加、質問種別変更、必須設定、セクション、分岐、ヘッダー画像、送信完了メッセージ。
- 回答画面: 回答進捗、必須/任意表示、セクション遷移、分岐、受付停止/期限切れ表示。
- JSON作成: LLMに書かせたIDなしJSONを貼り付けてフォームを作成する画面。

## 作成済みフォームから開ける機能

作成済みタブの各フォームには、仕様書用の遷移確認ボタンを用意しています。

- 回答: 回答一覧、回答詳細、CSVコピー、CSVダウンロードのモック。
- 回答者: 配布対象者一覧、未回答/回答済みフィルタ、CSV取り込みエラー例、一括再送導線。
- 通知: 回答依頼、期限前リマインド、未回答者リマインド、受付終了通知、通知文面プレビュー。
- 版管理: 公開中/下書き/過去版の履歴、現在版と前版の比較。
- 公開レビュー: 公開前チェック、質問数、対象者数、通知設定数、公開後状態の確認。
- 公開設定: 公開状態、公開範囲、回答リンク、回答受付ルール、共同編集者権限。
- 共有: 回答リンクの確認、コピー、回答画面を別タブで開く導線。
- プレビュー: 回答者視点のフォーム確認。
- 編集: 作成画面へ戻ってフォームを編集。

## ヘッダーから開ける仕様確認画面

- 監査ログ: フォーム公開、JSONインポート、CSV出力、通知送信、受付停止などの操作履歴を確認できます。
- 状態切替: 画面仕様書で必要になりやすい状態を切り替えて表示できます。

状態切替で確認できるパターン:

- 回答0件
- 権限なし
- 期限切れ
- 回答済みで再回答不可
- JSON取り込みエラー
- CSV取り込みエラー

## JSON作成機能

JSONボタンから、LLM向けのフォーム作成JSONを確認・貼り付けできます。内部IDは表示しません。

主なフィールド:

- `title`: フォーム名
- `description`: 説明文
- `settings`: 公開範囲、期限、受付状態、送信完了メッセージ
- `sections`: セクション名と説明
- `questions`: 質問一覧
- `branch`: 回答による分岐設定

質問タイプ:

- `shortText`
- `paragraph`
- `radio`
- `checkbox`
- `select`

## このモックで確認したい仕様論点

- 公開前に確認する項目は何か
- 公開後に編集した場合、版をどう扱うか
- 誰を回答対象者として管理するか
- 未回答者への通知タイミングはいつか
- 回答CSV出力やJSONインポートは監査ログに残すか
- 権限なし、期限切れ、空状態、インポート失敗時の画面文言をどうするか
