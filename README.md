# Google Form Clone Mock

Googleフォーム風のフォーム作成・回答管理モックです。バックエンドAPIには接続せず、画面仕様書や画面遷移図を作るために、フォーム作成から公開、回答、通知、監査ログ、エラー状態までをフロントだけで確認できるようにしています。

各機能の詳しい説明は [FEATURES.md](FEATURES.md) を参照してください。

## 起動方法

```bash
npm install
npm run dev
```

Viteの表示するURLをブラウザで開きます。ログイン画面はデモ実装のため、任意のメールアドレスとパスワードで進めます。

## 主な画面

- ログイン画面: メールアドレス入力、パスワード入力の2段階ログインモック。
- 管理トップ: テンプレート一覧、ホバー時プレビュー、回答中/作成済みタブ、作成済みフォーム検索、状態絞り込み、期限/更新日/名前順ソート。
- フォーム作成: 質問追加、質問種別変更、必須設定、セクション、分岐、質問表示条件、質問詳細設定、見た目、回答ルール、ヘッダー画像、送信完了メッセージ、下書き保存、自動保存、公開保存。
- 回答画面: 回答進捗、必須/任意表示、セクション遷移、分岐、質問表示条件、入力形式チェック、受付停止/期限切れ表示、1人1回回答、送信後編集、回答コピー送信表示。
- JSON作成: フォーム作成画面から、LLMに書かせたIDなしJSONを貼り付けてフォームを作成する画面。反映前に形式チェック、概要プレビュー、現在フォームとの差分を表示します。
- 権限切替: ヘッダーで管理者/編集者/閲覧者を切り替え、編集・保存・公開の可否を確認できます。
- デモデータリセット: 高度な操作メニューから、保存済みフォーム、回答、監査ログ、通知設定、版履歴を初期化できます。

## 作成済みフォームから開ける機能

作成済みタブの各フォームには、仕様書用の遷移確認ボタンを用意しています。

- 回答: 回答一覧、回答詳細、回答分析、CSVコピー、CSVダウンロードのモック。
- 回答者: 配布対象者一覧、未回答/回答済みフィルタ、CSV取り込みエラー例、一括再送導線。
- 通知: 回答依頼、期限前リマインド、未回答者リマインド、受付終了通知、通知文面プレビュー、通知ルール保存、テスト送信ログ。
- 版管理: 公開中/下書き/過去版の履歴、現在版と前版の比較、新しい下書き版の作成。
- 公開レビュー: 公開前チェック、質問数、対象者数、通知設定数、公開後状態の確認。
- 公開設定: 公開状態、公開範囲、回答リンク、回答受付ルール、共同編集者権限。
- 共有: 回答リンクの確認、コピー、回答画面を別タブで開く導線。
- プレビュー: 回答者視点のフォーム確認。
- 編集: 作成画面へ戻ってフォームを編集。
- 複製: 既存フォームを下書きとして複製。
- アーカイブ: 作成済みフォームをアーカイブ状態へ変更。
- 削除: ローカル保存されたフォームを一覧から削除。

## 保存と公開の扱い

このモックでは、下書き保存と公開保存を `localStorage` に保存します。保存したフォームは作成済みタブに追加され、共有リンクから回答画面を開けます。

- 編集中はヘッダー下に「未保存の変更あり」「保存済み」を表示します。
- 下書き保存すると作成済み一覧に下書きとして反映します。
- 公開保存すると公開前チェックの結果に応じて「公開中」または「公開前確認あり」として反映します。
- 公開前チェックに未完了項目がある場合は確認ダイアログを表示し、下書きへ戻るか、確認して公開するかを選べます。
- 公開前チェックはフォームタイトル、説明、質問数、必須質問、セクション紐づき、公開範囲、回答期限、対象者、通知、送信完了メッセージを確認します。
- 未保存の変更がある状態でリロードしようとするとブラウザの確認を表示します。編集画面から管理トップへ戻る時も確認します。

## 回答と監査ログの扱い

回答画面で送信した内容は `localStorage` に保存し、回答管理画面の回答一覧・回答数へ反映します。回答詳細では「確認済み」「差し戻し」への状態変更も確認できます。

次の操作は監査ログ画面に追加されます。

- 下書き保存
- フォーム公開
- JSONインポート
- 回答送信
- 回答CSVコピー/ダウンロード
- 回答ステータス更新
- フォーム複製/アーカイブ/削除
- 通知設定更新
- 通知テスト送信
- 版履歴追加

## 画面遷移図で確認する代表パターン

| パターン | 開始画面 | 主な遷移 | 確認する状態 |
| --- | --- | --- | --- |
| 新規作成 | 管理トップ | 新規フォーム作成 -> 編集 -> 下書き保存 -> 作成済み | 未保存表示、保存済み表示、作成済み一覧への反映 |
| 公開前エラー | 編集 | 公開として保存 -> 公開前チェック -> 下書きへ戻る | 必須チェック、公開ブロック、強制公開導線 |
| JSON作成 | 編集 | JSONインポート/出力 -> 差分確認 -> JSON反映 | IDなしJSON、エラー表示、差分表、監査ログ |
| 回答送信 | 回答URL | 回答入力 -> 送信完了 -> 回答管理 | 回答一覧反映、回答分析、回答済み制御 |
| 回答確認 | 作成済み | 回答 -> 回答詳細 -> 確認済み/差し戻し | ステータス更新、監査ログ |
| 通知設定 | 作成済み | 通知 -> ルール編集 -> 設定反映/テスト送信 | 通知保存、送信対象、監査ログ |
| 版管理 | 作成済み | 版管理 -> 新しい版を作成 -> 編集 | 版履歴追加、公開後編集の扱い |
| 権限差分 | ヘッダー | 管理者/編集者/閲覧者切替 | 編集不可、保存不可、公開不可 |
| 例外状態 | 状態切替 | 期限切れ/権限なし/回答済み/JSON失敗 | エラーメッセージ、戻り導線 |

## ヘッダーから開ける仕様確認画面

- 監査ログ: フォーム公開、JSONインポート、CSV出力、通知送信、受付停止などの操作履歴を確認できます。
- 状態切替: 画面仕様書で必要になりやすい状態を切り替えて表示できます。

状態切替で確認できるパターン:

- 回答0件
- 権限なし
- 期限切れ
- 回答済みで再回答不可
- JSON取り込みエラー
- CSV取り込みエラー

## JSON作成機能

フォーム作成画面のJSONボタンから、LLM向けのフォーム作成JSONを確認・貼り付けできます。内部IDは表示しません。

反映前チェックでは、次のような問題を検出します。

- `title` がない
- `sections` または `questions` が空
- 未対応の `type` を指定している
- 質問タイトルがない
- 質問の `section` が `sections` に存在しない
- 選択式なのに `options` が空
- `rating` の最小値が最大値以上
- `matrix` の行または列が空
- `file` の最大ファイル数が1未満

主なフィールド:

- `title`: フォーム名
- `description`: 説明文
- `settings`: 公開範囲、期限、受付状態、テーマ色、背景色、進捗表示、回答ルール、送信完了メッセージ
- `sections`: セクション名と説明
- `questions`: 質問一覧
- `branch`: 回答による分岐設定
- `visibilityCondition`: 他の質問の回答に応じて質問を表示する条件
- `validation`: メール/URL/電話番号形式、文字数、数値範囲、エラーメッセージ

編集画面で触れる主な項目:

- フォーム全体: テーマカラー、背景色、進捗バー表示、必須表示形式、送信ボタン文言。
- 回答ルール: メール収集、匿名回答、1人1回、送信後編集、回答コピー送信。
- 質問詳細: プレースホルダー、回答例、説明文表示、回答形式、文字数/数値範囲、エラーメッセージ。
- 選択肢: 一括追加、並び替え、「その他」、ランダム表示、選択肢ごとの分岐先。
- 表示条件: 選択式の回答に応じて、特定の質問だけを表示。
- 編集効率: 質問の複製、上下移動、ドラッグ並び替え。

質問タイプ:

- `shortText`
- `paragraph`
- `email`
- `number`
- `date`
- `time`
- `radio`
- `checkbox`
- `select`
- `rating`
- `matrix`
- `file`
- `consent`

## このモックで確認したい仕様論点

- 公開前に確認する項目は何か
- 公開後に編集した場合、版をどう扱うか
- 誰を回答対象者として管理するか
- 未回答者への通知タイミングはいつか
- 回答CSV出力やJSONインポートは監査ログに残すか
- 権限なし、期限切れ、空状態、インポート失敗時の画面文言をどうするか

## 注意点

このリポジトリは画面仕様書・画面遷移図用のフロントエンドモックです。実運用に必要なサーバー処理は含めていません。

- 保存先はブラウザの `localStorage` です。別ブラウザ、別端末、シークレットウィンドウには同期されません。
- ログインはデモです。入力したメールアドレスを表示に使うだけで、認証や権限判定は行いません。
- 権限切替はUI確認用です。実際の認可、サーバー側のアクセス制御、リンク共有制御はありません。
- 1人1回回答、送信後編集、回答コピー送信はフロント上の状態確認用です。実際のメール送信、ユーザー単位の厳密な重複防止、サーバー側の回答保存はありません。
- ファイル設問はファイル名をフォーム上で扱うだけです。アップロード、ウイルスチェック、容量制限、サーバー保存はありません。
- 回答者CSV取り込みは仕様確認用のサンプル表示です。実際のCSVパースや配布対象者の永続更新はありません。
- 通知設定、監査ログ、版履歴は `localStorage` に保存されますが、実際のメール送信やサーバー側履歴保存はありません。
- JSONチェックは画面仕様用の簡易検証です。実運用ではサーバー側でもスキーマ検証、サイズ制限、権限チェック、監査ログ記録が必要です。
- 編集画面、管理画面、回答画面、JSON画面は遅延読み込みしています。ビルド時のチャンクサイズ警告は解消済みです。

```bash
npx eslint src/features/form-builder
npm run build
npm run test:e2e
npm audit
```

E2EテストはPlaywrightを使います。初回はブラウザ本体が未インストールの場合があるため、その場合は次を実行してください。

```bash
npx playwright install chromium
```
